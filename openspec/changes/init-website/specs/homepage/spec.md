## ADDED Requirements

### Requirement: Project Overview
The homepage SHALL display a concise description of Archi — its purpose as a retrieval-augmented generation framework for research and education teams, its deployment at CMS, and its open-source nature.

#### Scenario: First-time visitor reads overview
- **WHEN** a visitor lands on the homepage
- **THEN** they see the Archi logo, project name, and a summary paragraph describing the project
- **AND** a link to the GitHub repository is visible

### Requirement: Key Capabilities Section
The homepage SHALL list Archi's key capabilities including customizable AI pipelines, data ingestion connectors, multiple interfaces, LLM support, modular design, and containerized deployment.

#### Scenario: Capabilities are scannable
- **WHEN** a visitor scrolls the homepage
- **THEN** they see a section with key capabilities presented as a concise list or card grid

### Requirement: Quick Links
The homepage SHALL provide prominent links to the project documentation, GitHub repository, and latest news.

#### Scenario: Visitor finds documentation
- **WHEN** a visitor wants to learn more about using Archi
- **THEN** they can click a clearly labeled link to the documentation site (archi-physics.github.io/archi/)

### Requirement: Institutional Affiliation Display
The homepage SHALL visually indicate the project's institutional affiliations (MIT, CERN, Fermilab) through logos or named references.

#### Scenario: Institutions are visible
- **WHEN** a visitor views the homepage
- **THEN** they see the names or logos of affiliated institutions
